## [1.1.0] - 2024-12-25

### 🎉 重大更新

这是一个重要的功能增强版本，引入了 GitHub Actions 自动化和多项用户体验改进。

### ✨ 新增功能

#### CI/CD 自动化
- GitHub Actions 自动构建 Docker 镜像
- 自动推送到 GitHub Container Registry (ghcr.io)
- 支持多架构构建 (linux/amd64, linux/arm64)
- 自动创建 GitHub Release
- 自动打包源码 zip 文件
- 版本标签自动管理 (latest, vX.X.X, main)

#### 下载体验优化
- **取消下载即时反馈**: 点击取消按钮立即弹窗提示，并更新界面状态
- **智能文件名处理**: 超长文件名自动优化，提取关键信息
  - 视频文件智能识别标题、年份、季集号 (S01E05)
  - 自动过滤广告关键词和无用信息
  - 文件名长度控制在 40-50 字符
  - 显示优化提示信息
- **增强进度显示**:
  - 进度条缩短 50% (10字符 vs 20字符)
  - 显示正在下载的文件名
  - 实时显示下载速度 (MB/s 或 KB/s)
  - 智能预测剩余时间 (秒/分秒/时分)

#### 部署方式改进
- 支持直接从 GitHub Container Registry 拉取镜像
- 无需本地构建，开箱即用
- 更新 docker-compose.yml 支持 GHCR
- 提供完整的 GHCR 使用文档

### 🔄 变更

#### 文件名处理算法

  - 改进进度更新机制

#### 用户交互
- 改进取消操作反馈机制
- 优化重复文件提示信息
- 增强错误处理和日志输出

### 📦 部署更新

## [1.0.0] - 2024-12-25

### 🎉 初始发布

这是 TeleFlux 的首个正式版本,提供完整的 Telegram 文件自动下载功能。

### ✨ 新增功能

#### 核心功能
- 自动接收并下载 Telegram 文件
- 智能文件分类(音乐/视频/其他)
- 可自定义保存路径
- Docker 容器化部署
- Docker Compose 支持

#### 文件处理
- 智能文件名清理和格式化
  - 音乐文件: "歌名-歌手" 格式
  - 视频文件: 关键词提取(超过7字自动精简)
  - 移除特殊字符和广告信息
- 重复文件智能检测
  - 提供覆盖/加序号/取消三个选项
  - 避免文件冲突
  - 保留原文件选项

#### 下载管理
- 实时进度显示
  - 图形化进度条
  - 显示下载百分比
  - 显示文件大小和已下载大小
- 下载控制
  - 暂停/继续下载
  - 取消下载(自动清理临时文件)
- 无文件大小限制
- 支持大文件下载(使用 MTProto)
- 断点续传功能
  - 支持暂停后继续
  - 支持容器重启后恢复

#### 用户体验
- 友好的中文界面
- 清晰的按钮标签("加序号"代替"保留原文件并加序列号继续下载")
- 详细的状态反馈
- 多种交互选项

#### 技术特性
- 基于 Telethon 库
- Python 3.11+ 支持
- 异步 I/O 处理
- 高效的文件流处理
- 缓存文件本地保存

### 🔧 配置选项

- `API_ID` - Telegram API ID (必需)
- `API_HASH` - Telegram API Hash (必需)
- `BOT_TOKEN` - Telegram Bot Token (必需)
- `MUSIC_PATH` - 音乐文件保存路径 (可选,默认: /data/Music)
- `VIDEO_PATH` - 视频文件保存路径 (可选,默认: /data/Video)
- `DOWNLOAD_PATH` - 其他文件保存路径 (可选,默认: /data/Download)
- `CACHE_PATH` - 缓存文件路径 (可选,默认: /app/cache)

### 📦 部署方式

- Docker Run 支持
- Docker Compose 支持
- 环境变量配置
- Volume 数据持久化

### 📝 文档

- 完整的 README.md
- 详细的安装步骤
- Docker 部署指南
- 故障排查指南
- 配置说明文档

## 版本说明

### 版本号格式

采用语义化版本: `主版本号.次版本号.修订号`

- **主版本号**: 不兼容的 API 修改
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 变更类型

- `新增` - 新功能
- `变更` - 对现有功能的变更
- `弃用` - 即将移除的功能
- `移除` - 已移除的功能
- `修复` - 任何 bug 修复
- `安全` - 安全相关的修复
